新ubuntu桌面版日志

旧电脑上面安装ubuntu桌面版，可以当作服务器用，外网可以访问，是通过路由器设置的，并且设置为了DMZ主机。安装了nps，与笔记本电脑上的npc连接，实现外网使用远程桌面访问。安装jupoyterlab，网页版vscode，docker，nextcloud等。云帮手安装探针查看电脑资源，管理进程。花生壳内网穿透，实现ip发生了变化，可以通过花生壳登录来拆看ip。还有闭上盖子不休眠，关闭图形界面，更换镜像源好像没有写上去。看来一下，之前的的日志写到了。感觉这篇博客还是很全面的，介绍了我安装一个ununtu桌面版当作服务器用会做什么。

2021.1.25

真是没事找事，我看根目录下看见了几个不知道怎么来的文件夹，进入以看全是空的，就删除。结果删除了/lib，ls命令都不能用了。还是重装一下吧，我怕又装不好，但没有出现那样的问题，很顺利。正好可以重新设置密码，重新写操作日志，也删除了不知都有没有用的文件。就是觉得自己手很贱，自己接受不了文件夹很乱的样子。系统之前的是20.04的，要是直接安装mysql又有问题

  • 安装ssh服务

要电脑重启一下才连的上

  • 安装vim

  • 设置合上盖子不休眠

2021.1.26

  • 笔记本电脑上面连接nps，成功，真的很流畅，感觉真棒。
  • 笔记本电脑上面安装服务端，旧电脑上安装客户端，连接后查看。我是开放笔记本电脑上的一个端口用于连接，结果看到的是127.0.0.1不是真实IP，因为自己是端口转发的，失败。
  • 然后自己又看了一下 云帮手的文档，原来是在服务器端安装脚本，之前看了一下以为是要输入ip地址来管理。现在知道是登录账号，安装脚本，然后就可以监控了。
  • 云帮手确实好用，但是我安装在我旧电脑上面是出错了，探针启动出错，安装很友好的，软件提供了一条命令直接复制运行就可以了。我以为是自己系统原因，我在我目前的两台服务器ubuntu20和win2012都安装成功了，然后我在docker中安装失败了。然后我重启旧电脑还是出现哪个问题，放弃感觉很舍不得，自己都准备再重新安装一下系统，但是不划算，然后又搜索阿里云监控，安装了阿里云探针，可以监控网络，CPU和内存，竟然看不了公网ip。云帮手的博客很少，官方提供的文档很好但是没出现我这样的问题。我只有顺着错误提示去解决，但是还是没成功。我在重启电脑，再试一下，从右下加的通知弹出了连接lxx-Device的提示，那时候感觉nice，不知道什么原因。
  • 安装好后，不能访问nps和宝塔，我关闭防火墙后就可以访问了
#查看防火墙状态
sudo ufw status

#开启防火墙
sudo ufw enable

#关闭防火墙
sudo ufw disable
  • 宝塔进入重启才能运行
    /etc/init.d/bt restart

查看内存

sudo apt purge snapd

  • 还是使用1.1版本吧，密码没设置成功，安装好了直接删掉没用的

  • 我擦，踩坑，什么密码，我试了半天，原来是教程错误，不是再ipython上运行生成哈希密码再填入配置文件，而是输入

  • 删除django，hexo

2020.1.27

配置git

2020.1.30

  • 删除了云帮手，今天还以为旧电脑出硬件了问题，这给自己提了个醒，觉得自己还是需要自己的云服务器，也准备搞个弹性计算的服务器了，可以看ip，用家里的电脑转发流量。

改变

  • 改变注意了，不想再重新迁移一遍了，虽然物理损坏的风险是有的，重要的东西就是代码，我每次写代码养成一个上传的习惯，这样就没有事了。网盘里面的东西也备份了。
  • 于是我就在寻找是否有个像 云帮手这样的软件，可以在服务器安装探针，然后客户端就可以监控状态，也可以看到公网ip。阿里云云监控可以看到状态，看看不到公网IP，腾讯云云监控就坑了，竟然不能添加费腾讯云服务器，真的是要着有何用。
  • 再尝试了 云锁，这个和云帮手差不多，但是功能实在是太少了，而且收费，云帮手真的是业界良心。但是我只需要看ip，这个也可以看ip，但是我重启路由器，IP地址发生了变化，我再去刷新，云锁上面的ip还是没有变，重启电脑后再看也是没有变，对这个希望很大的，但是失败了。现在去卸载中
  • 云锁：http://help.yunsuo.com.cn/guide/install/linux.html
# 卸载
/usr/local/yunsuo_agent/uninstall
  • 然后我认识了 小皮面板，和宝塔面板功能和内容差不多，感觉小巧精致一些，要是没有安装宝塔我就使用这个。
  • 服务器监控工具不知道再用啥了，其他的感觉差不多，看界面比较老就没有再试试了。然后看看路由器功能里面找找答案，认识了 DMZ主机，看看 DDNS 没有什么用，这个可以提供域名，然后登录，但是路由器重启需要登录路由器再登录域名。上面还有花生壳提供的域名，但是需要买算虽然1不是很贵，但是我不知道是不是我需要的，然后看到了一篇博客，大致意思说花生壳域名也要那样登录http://bbs.oray.com/thread-64835-1-1.html
  • 我竟然看到了tp-lnk可以ssh登录，就是提到的，两篇，没什么用，也没看懂写的什么。https://www.cnblogs.com/SilenceRet/p/9238116.html
  • 花生壳，向日葵什么的都试了的，就花生壳是内网穿透，但是没有设备，我就以为要买硬件 花生盒子，花生棒才能使用，就像登录路由器一样，但是使用它的硬件可以用这个软件再外面登录管理家里的路由。
  • 再想到带宽拨号，没有公网IP真是伤，我猜想拨号登录可以再联通哪里登录，然后可以查看分配的ip。需要身份证号码+地区，估计是老爸的，我不想去试了，真的没有别的办法，我当时想，没有什么东西再服务器上下载运行，然后其他地方可以查看到ip吗？
  • 最后没有办法我准备等学习shell，或者用python写代码，ip改变发送邮件。

  • 对了，哪个 云帮手探针添加成功了，主要用来管理进程，查看资源。

云帮手原因

  • 查看了一下安装日志，上面说服务启动失败，提示说systemctl status cxAgentd.service，就是叫我查看服务状态
  • 然后我在官方文档查看启动命令是/etc/init.d/cxAgentd start ，输入却提示没有这个目录
  • 然后我又先到了，之前看到一篇博客说他不喜欢ubuntu18，应为从18开始，服务就是用systemctl管理，于是我就输入
systemctl start cxGuardd.service
  • 什么都没提示，也就是说明没错误，在看电脑上弹出了连接成功的通知
  • 学会看日志来解决错误真的很重要，自己的积累也很重要
